# 贡献奖励与支持

如果你为 **nexu** 做出贡献，我们希望这件事不只是"PR merge 了"，还包括：

- 你的每一次认真投入，都能被看见
- 你的贡献，不会只停留在 merge 记录里
- 你的成长，会被持续记录和积累
- 你第一次参与时，也能感受到社区的欢迎

## 我们会提供什么帮助

对于 `good-first-issue` 和首次贡献者，我们会尽量提供这些支持：

- **更清晰的题目范围**：尽量把题目拆成小而清晰、可验收的任务
- **更明确的 review 反馈**：如果需要修改，我们会尽量说明原因和方向
- **首次贡献引导**：第一次参与时，尽量给你必要的说明和建议
- **公开致谢与积分记录**：合入后会进入贡献者致谢与积分流程

如果你还没开始，可以先看：

- [第一次提 PR 指南](/zh/guide/first-pr)
- [参与贡献](/zh/guide/contributing)
- [Good First Issue 列表](https://github.com/nexu-io/nexu/labels/good-first-issue)

## 你的贡献会展示在哪里

如果你的贡献被合入，后续会展示在这些位置：

1. **GitHub README**：展示贡献者，并同步呈现 **日榜、月榜和历史累计榜单**
2. **Discord [Start your first Nexu PR](https://discord.gg/vMrySTJW8u)**：Bot 自动发布排行榜动态

## 当前积分规则

积分按 Issue 标签分为两个档位，简单明确：

| Issue 标签 | 积分 | 等值额度 |
| --- | --- | --- |
| `help wanted` | 2,000 | ≈ $20 USD nexu 使用额度 |
| `good first issue` | 500 | ≈ $5 USD nexu 使用额度 |
| 其他（无以上标签） | 500 | ≈ $5 USD nexu 使用额度 |

### 发放流程

1. **贡献者提交 PR** 并关联对应 Issue
2. **研发团队审核** PR 的代码质量和完成度
3. **PR 合并后**，由团队确认积分并发放到你的账户

> 积分的最终发放以研发团队审核通过并合并 PR 为准。团队保留对积分发放的最终解释权。

## 积分可以做什么

积分可直接兑换为 **nexu 产品使用额度**，也是你在社区中的贡献凭证：

- 积分 = 真实产品使用额度，贡献即获得回报
- 首次贡献者能获得明确的正反馈
- 长期贡献者在社区里积累可见身份
- 积分记录沉淀为你的开源贡献履历
